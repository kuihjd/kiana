# chat_forward

群聊消息打包插件。支持将群聊中最近的消息打包成合并转发消息，方便保存和分享聊天记录。

## 功能特性

- 打包群聊最近的消息为合并转发消息
- 支持自定义打包条数
- 自动过滤已撤回和无效消息
- 群组冷却机制，防止滥用
- 支持分群开关控制

## 命令列表

| 命令 | 说明 | 权限 |
|------|------|------|
| `打包消息` | 打包最近 15 条消息（默认） | 所有人 |
| `打包消息 [条数]` | 打包指定条数的消息 | 所有人 |
| `打包记录` | 同 `打包消息` | 所有人 |
| `打包记录 [条数]` | 同 `打包消息 [条数]` | 所有人 |

## 使用示例

```
用户: 打包消息
Bot: [合并转发消息]
     ├── 用户A: 消息1
     ├── 用户B: 消息2
     ├── 用户C: 消息3
     └── ...

用户: 打包消息 50
Bot: [合并转发消息（包含50条消息）]

用户: 打包记录 30
Bot: [合并转发消息（包含30条消息）]
```

### 冷却提示

```
用户: 打包消息
Bot: 冷却中，请等待 8 秒
```

## 配置项

在 `.env` 或 `.env.prod` 中配置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `chat_forward_plugin_enabled` | bool | true | 是否启用打包消息插件 |
| `chat_forward_group_mode` | str | "all" | 群组控制模式：all/whitelist/blacklist |
| `chat_forward_group_whitelist` | list | [] | 白名单群组 |
| `chat_forward_group_blacklist` | list | [] | 黑名单群组 |
| `chat_forward_default_count` | int | 15 | 默认打包消息条数 |
| `chat_forward_max_count` | int | 100 | 最大允许打包消息条数 |
| `chat_forward_cooldown` | int | 10 | 冷却时间（秒） |

## 注意事项

- 仅支持群聊使用
- 触发命令的消息本身不会被打包
- 已撤回的消息不会被打包
- 每个群组有独立的冷却时间
- 打包条数不能超过配置的最大值（默认 100 条）
- 条数必须大于 0
